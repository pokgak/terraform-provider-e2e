# e2e_container_registry (Resource)

Provides an E2E Container Registry resource.

This resource allows you to provision and manage Container Registry projects on your E2E project. When applied, it creates a new project. When destroyed, it is removed.

<!-- schema generated by tfplugindocs -->
## Example Usage

```hcl
resource "e2e_container_registry" "example" {
  project_id    = "your-project-id"
  location      = "Delhi"
  project_name  = "my-registry-project"
  prevent_vul   = true
  severity      = "high"
}
```

## Schema

### Required

- `project_id` (String) The project ID to associate the Container Registry with.
- `location` (String) The location/region for the Container Registry.
- `project_name` (String) The name of the Container Registry project.

### Optional

- `prevent_vul` (Boolean) Whether to prevent vulnerable images (default: false).
- `severity` (String) The severity level for vulnerabilities (default: "low"). Allowed values: `"low"`, `"medium"`, `"high"`, `"critical"`.

### Read-Only

- `id` (String) ID of the Container Registry project.
- `setup_status` (String) The setup status of the Container Registry (e.g., "CREATED").
